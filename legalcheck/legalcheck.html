Paper Curation 적법성 검토

대륙아주 법무검토의견서(2026.2.27) 기준 · Local / Web Deploy 분야별 리스크

근거: 「AI를 이용한 논문 서지정보·초록·원문 활용 관련 법률자문」(전 36면) · 작성일 2026-07-20

전제 및 면책

본 문서는 법률자문이 아니라, 위 의견서의 결론을 paper-curation / paper-curio의 실제 동작(수집 경로·배포 payload·원문 처리·라이선스 게이팅)에 매핑한 기술·정책 검토 보고서이다. 신호등은 "이 의견서 기준의 상대적 리스크"를 나타내며 확정 판단이 아니다. 의견서 자체가 "구체적 사실관계가 달라지면 결과는 달라질 수 있다"고 명시하고 있고, 최종 판단은 법무·변호사의 몫이다.

🟢 리스크 낮음 · 적법 가능성 높음 🟡 조건부 · 주의 🔴 리스크 높음 · 금지

핵심 요약 — 안전 · 주의 · 금지

안전 — 제한 없이 수행 가능

  • 로컬(비공개) 이용 전반 — paper-curio(Zotero 플러그인), paper-curation Local(ai4s·scisci·smart-laboratory + serve_local). 내부·비영리이고 공중 전송이 없으므로 리뷰·figure·audio·Deep Research를 제한 없이 사용한다.
  • 본인 다운로드·API(arXiv·Semantic Scholar·OpenAlex)·오픈액세스 PDF에 기반한 수집과, 라이선스·약관을 준수하는 범위의 저장.
  • CC·CC0 라이선스 논문의 figure·리뷰 공개(출처표시 하).
  • 출처·라이선스·AI 생성 고지를 갖춘 오픈액세스(arXiv·OpenReview) AI 리뷰의 공개.
  • ND 라이선스 논문의 최소 페이지(제목·라이선스·원문 링크) 공개.

주의 — 조건 충족 시에만 · 운영 정책으로 관리

  • 공개(Web)되는 AI 리뷰·audio는 변형적 이용 논거가 있으나 확정 안전지대가 아니다. 대상을 오픈액세스로 한정하고, 원문 전문을 배제하며, 원문 링크를 제공하는 조건을 유지한다.
  • 공개 전송(④)은 원칙적으로 이용허락 대상이다. 공개 payload를 원문이 아닌 AI 리뷰로 한정한다.
  • arXiv·OpenReview figure는 공정이용 스탠스로 공개하되, PC_FIGURE_POLICY=strict에서는 게이팅한다.
  • 영리성/NC(⑧)와 제출원고(⑩)는 코드가 아닌 운영 정책으로 관리한다 — 무상 공개 유지, 제출원고 이용 시 사전 동의 확보.

금지 — 해서는 안 되는 것

  • 웹스크래핑을 통한 수집(저작권·데이터베이스제작자권 침해 리스크).
  • 원문 전문(text.md)의 공개 배포.
  • ND 라이선스 논문의 AI 리뷰·audio(2차적 저작물) 공개.
  • publisher(All rights)·unknown 라이선스 figure의 공개.
  • 서비스·결과물의 유상화·수익화(NC 위반 소지).
  • 오픈액세스(arXiv·OpenReview)가 아닌 논문을 공개 인덱스에 포함.
  • 출처·라이선스·AI 생성 고지를 누락한 상태의 공개.
  • 제출승인·실적증빙 원고를 사전 동의 없이 AI로 처리.

두 배포 모드 (배포 경계 아키텍처)

  • Local(비공개) — paper-curation 로컬 모드(ai4s·scisci·smart-laboratory, serve_local.py) + paper-curio(Zotero 플러그인). 게이팅이 없으며 리뷰·figure·audio·Deep Research가 전부 동작한다. 의견서상 내부·비영리 이용은 공정이용 범위에 들어 제약이 불필요하다.
  • Web deploy(공개) — Cloudflare(paper-curation.jehyunlee.dev), arXiv·OpenReview 오픈액세스 3개 토픽(humanoid·physical-ai·ai4s-icml2026). 원문 전문(text.md)은 배포 단계에서 하드 차단한다.
  • 게이팅 시점 — 생성기는 항상 전체(full)로 렌더한다(로컬용). prepare_deploy.py가 업로드 직전 배포 사본만 PUBLIC 모드(PC_PUBLIC_BUILD=1)로 재렌더해 라이선스 게이팅을 적용한 뒤 업로드하고, 업로드 후 로컬 전체 렌더를 자동 복원한다(API 키 복원과 동일 패턴). 즉 "로컬 자유 / 공개 안전"으로 분리한다.

공개본 라이선스 게이팅 정책

논문별 license는 OpenAlex(DOI/arXiv, 미해결분은 zotero_meta의 OpenReview URL)로 판별해 review.md frontmatter에 기록한다. 정책은 pipeline/lib/license_util.py가 단일 소스이다.

라이선스Figure 공개AI 리뷰·audio 공개비고
CC BY / BY-SA / BY-NC / BY-NC-SA / CC0허용허용모든 CC는 변경 없는 원본 재배포를 허용하며, NC는 비영리 사이트이므로 준수한다.
CC BY-ND / BY-NC-ND (ND)허용차단ND는 2차적 저작물(= AI 리뷰)을 금지하므로 리뷰·audio를 공개하지 않고, 최소 페이지와 원문 링크만 제공한다.
arXiv / OpenReview (OA 프리프린트)기본 허용허용공정이용 스탠스(비영리·출처표시·원문링크). PC_FIGURE_POLICY=strict이면 figure도 게이팅한다.
publisher(All rights) / unknown게이팅허용재배포 근거가 없어 figure를 공개하지 않고 캡션만 노출하며, 리뷰는 변형적 공정이용 스탠스로 유지한다.

의견서 판단 5축 (요약)

  • 수집 — 출판사 다운로드·API는 적법하며, 웹스크래핑은 저작권·데이터베이스제작자권 침해 리스크가 있다.
  • AI 분석·학습 — 내부·비영리·변형적이면 공정이용 가능성이 있다(제35조의5).
  • 저장(복제) — 라이선스·약관·CC를 준수하면 적법하다.
  • 전송·공유 — 사적이용복제(제30조)는 전송에 적용되지 않으므로 원칙적으로 이용허락이 필요하다.
  • 표시 의무 — 제37조 출처명시, CC BY 저작자표시, 인공지능기본법 제31조 AI 생성 고지가 요구된다.

분야별 리스크 표

분야 Local Web deploy 근거 · 설명
① 논문 수집 🟢 🟢 Zotero 본인 다운로드와 arXiv·Semantic Scholar·OpenAlex API, 오픈액세스 PDF만 사용하며 웹스크래핑을 하지 않는다(의견서 1.가·다). 공개 대상은 arXiv·OpenReview 오픈액세스이므로 근거가 더 명확하다. 기관 라이선스·약관이 AI 이용을 금지하는 논문은 대상에서 제외한다(2.D).
② 저장(복제) 🟢 🟢 저장은 복제에 해당하나 라이선스·약관·CC를 준수하고 적법하게 취득한 범위이면 합법이다(4.A). 원문 전문(text.md)은 배포 단계에서 하드 차단되어 Cloudflare 공개본에 포함되지 않는다.
③ AI 분석·리뷰 생성 🟢 🟡 내부·비영리·변형적 이용은 공정이용 가능성이 있다(2.A~C). 로컬은 의견서가 검토한 "본건 서비스" 자체에 해당한다. 공개는 변형적 논거가 유지되나 결과물을 불특정 다수에 공개하므로 "원내 한정" 전제를 벗어나며, ND 논문은 ⑨에서 별도 차단한다.
④ 전송·공유 🟢 🟡 제30조 사적이용복제는 전송에 적용되지 않는다. 로컬 열람·curio는 공중 전송이 없다. 공개는 공중 전송이므로 원칙적으로 이용허락이 필요하며, ⓐ payload가 원문이 아닌 AI 리뷰이고 ⓑ 대상이 이미 공개된 오픈액세스 프리프린트이며 ⓒ 원문 링크를 제공해 시장을 대체하지 않는 구조로 리스크를 완화하나 확정 안전지대는 아니다.
⑤ Figure 원본 재현 🟢 🟢~🟡 라이선스 기준으로 공개를 자동 게이팅한다. CC·CC0는 재배포가 허용되어 출처표시 하에 공개하고, publisher·unknown figure는 공개하지 않으며 캡션만 노출한다. arXiv·OpenReview는 공정이용 스탠스로 유지하되 strict 모드에서 게이팅할 수 있다. 로컬은 항상 전체를 렌더한다.
⑥ 출처·라이선스 표시 🟢 🟢 저자·DOI·arXiv 링크와 원저작권 귀속에 더해 CC 라이선스 종류를 라벨로 표시(예: "CC BY-NC-ND")하여 제37조와 CC BY 저작자표시 요건을 충족한다. 전 페이지에 적용한다.
⑦ AI 생성 고지 🟢 🟢 전 페이지 배너와 푸터에 AI 생성 고지를 표시한다(인공지능기본법 제31조). 로컬도 동일 생성기를 사용하므로 문구가 포함된다.
⑧ 영리성 / NC 🟢 🟡 로컬은 내부·비영리로 NC를 준수한다. 공개는 무상 제공인 한 대체로 NC를 준수하나, 유상 용역·수익과 결부되면 NC 위반 소지가 있다(21~22p, 31~32p). 운영 정책으로 관리한다.
⑨ ND(변경금지) 🟢 🟢 공개 ND 논문은 AI 리뷰(2차적 저작물)를 공개하지 않고 제목·라이선스·원문 링크의 최소 페이지만 노출하며, audio overview와 리뷰 텍스트 임베드를 제거한다. 로컬은 전체를 유지한다.
⑩ 제출원고 목적외 사용 🟡 🟡 실적증빙·제출승인용 원고를 목적을 벗어나 AI에 사용하면 공정이용 가능성이 낮다(2.E). paper-curation은 arXiv·공개 논문 기반이라 해당 사례가 적으며, 사전 동의서로 관리한다.

종합 신호등 (제품·모드별)

paper-curio (Zotero 플러그인, 완전 로컬) — 🟢 가장 안전

사적이용복제(제30조)에 해당하고 공중 전송이 없다. 의견서가 권장한 "개인 기기 저장" 설계에 부합한다.

paper-curation Local (ai4s·scisci·smart-laboratory + serve_local) — 🟢

의견서의 "본건 서비스"에 정확히 대응한다. 게이팅 없이 리뷰·figure·audio·Deep Research를 전부 사용하며, 내부·비영리로 공정이용 범위에 든다.

paper-curation Web deploy (humanoid·physical-ai·ai4s-icml2026) — 🟢~🟡

arXiv·OpenReview로 대상을 한정하고 원문을 배제하며, 출처·CC 라이선스·AI 고지를 표시하고 figure·ND·audio를 라이선스 기준으로 게이팅한다. 잔여 리스크는 공개 전송의 원리적 성격(③·④)과 영리성(⑧)·제출원고(⑩)이며, 이들은 운영 정책 영역에서 관리한다.

구현된 통제

  • AI 생성 고지 + 출처/원저작권 문구 — 생성기 소스(review_to_html.py·build_topic_index.py)에 반영되어 매 빌드에서 자동 재현된다.
  • 라이선스 자동 판별 — backfill_licenses.py가 OpenAlex(및 zotero_meta의 OpenReview URL)로 논문별 license를 frontmatter에 기록한다(배포 3토픽 2337편). 정책 코어는 lib/license_util.py이다.
  • 공개본 라이선스 게이팅 — 라이선스 라벨 표시(⑥), figure 게이팅(⑤: publisher·unknown 미공개), ND 리뷰·audio 차단(⑨: 최소 페이지 + 원문 링크).
  • 배포 경계 아키텍처 — prepare_deploy.py가 업로드 직전 배포 사본만 PUBLIC 모드로 재렌더하고, 업로드 후 로컬 전체 렌더를 복원한다. 로컬은 무제한을 유지한다.
  • 배포 범위 고정 — 배포 대상을 3개 토픽으로 한정한다(.assetsignore로 smart-laboratory/·과대 PDF·_agent/ 제외). 원문 전문은 공개에서 제외한다.

운영 정책으로 관리하는 항목

  • ⑧ NC · ⑩ 제출원고 — 코드가 아니라 운영 정책으로 관리한다: 서비스·결과물의 유상화를 금지하고, 제출원고를 AI에 이용할 때 사전 동의서를 확보한다.
  • unknown 라이선스 — zotero_meta에 매칭되지 않는 소수 논문은 unknown으로 두어 figure 게이팅 대상(안전측)으로 처리한다.
  • 비배포 토픽 — ai4s·scisci는 공개 인덱스에 연결되지 않는다. 전체 적용이 필요하면 python pipeline/backfill_licenses.py(토픽 미지정)로 커버한다.
  • 정책 토글 — PC_FIGURE_POLICY=strict(arXiv·OpenReview figure까지 게이팅), PC_ND_POLICY=show(ND 억제 해제).
